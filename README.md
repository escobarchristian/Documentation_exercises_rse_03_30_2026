# Documentation Exercises

Hands-on exercises for learning how to document research software. Students practice writing docstrings, READMEs, and constructive code review comments — all inside Google Colab with no setup required.

## Exercises

### Exercise 1 — Fix the Mystery Functions

Two short Python functions with bad names and no documentation. Figure out what each one does, then rewrite it with a descriptive name, type hints, and a Google-style docstring.

**What you'll practice:** naming, type hints, docstring structure, `help()`

### Exercise 2 — Write a README

Fill in a README template for a simple project (a grade calculator script). The structure is provided — you fill in the content.

**What you'll practice:** README sections (purpose, installation, usage, license), Markdown syntax

### Exercise 3 — Code Review Practice

Review a short function with common issues and write three constructive comments using the sentence starters provided.

**What you'll practice:** identifying documentation gaps, writing feedback that is constructive and specific

## Getting Started

1. Download the notebooks and upload them to your Google Colab
2. Work through the exercises in order — each one builds on the previous
3. Solutions are hidden at the bottom of each notebook; try before you peek

## Prerequisites

- Basic Python knowledge (functions, lists, loops)
- A Google account for Colab access
- No installation needed
